# Contributing to Governed IaC Reference

Thank you for your interest in contributing!

## How to Contribute

- All changes must be proposed via pull request (PR) to the `main` branch.
- No direct commits to `main` are allowed.
- All commits must be GPG signed.
- PRs must pass all required status checks (CI, security, policy, lint).
- At least one maintainer must approve each PR.
- Code owner review is required for protected areas (e.g., infra/, policies/).

## Commit Guidelines

- Use clear, conventional commit messages (e.g., `feat:`, `fix:`, `docs:`).
- Group related changes in a single commit when possible.
- Sign all commits with your GPG key.

## Code of Conduct

- Be respectful and constructive in all interactions.
- Report any unacceptable behavior to the maintainers.

## Reporting Issues

- Use GitHub Issues for bug reports, feature requests, and questions.
- Provide as much detail as possible for faster resolution.

## Governance

- See GOVERNANCE.md for authority, review, and exception processes.

---

We appreciate your contributions to a secure, auditable, and governed IaC ecosystem!
