# Security Policy

## Reporting Vulnerabilities

If you discover a security vulnerability in this repository, please report it as soon as possible:

- Email: [reuben@hummbl.io](mailto:reuben@hummbl.io)
- Or open a confidential security advisory via GitHub (recommended)

Please include as much detail as possible to help us reproduce and address the issue quickly.

## Supported Versions

Only the latest version of the repository is actively supported and receives security updates.

## Responsible Disclosure

We request that you give us a reasonable time to investigate and remediate any reported vulnerabilities before public disclosure.

## Security Best Practices

- All code changes are reviewed and must pass security scans (TFLint, Checkov, Trivy)
- Branch protection and signed commits are enforced
- Policy as code is used to codify security and compliance requirements

Thank you for helping us keep this project and its users secure!
